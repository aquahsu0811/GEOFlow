# yao-geo-knowledge-base-builder

## 作用

基于官网、产品页、帮助中心、白皮书、销售材料、媒体稿和资质文件生成 GEO 品牌知识库、完整实体清单、真实数据获取边界、事实卡、FAQ、禁用表达、Prompt 输入包和来源索引。

## 适合什么

- 内容生成、内容改造、页面设计、监测和客服问答前置准备。
- 面向 Kimi、千问、DeepSeek、豆包、元宝复用结构化品牌事实。
- 需要中文简体 Word、PDF、HTML、Markdown 四件套交付。

## 输出结构

- 系统化结构知识库：结构化摘要、基础信息、定位、产品矩阵、指标、案例/社会证明、时间线、差异化、查询词和表达规范。
- 完整实体清单：品牌、公司、产品、服务、AI 能力、价格、指标、地区、渠道、竞品、来源和待确认边界。
- 数据获取层：标明本次使用公开网页、用户文件、授权私有源、纯 brief 还是不可访问源，并给出新鲜度风险。
- GEO 复用层：品牌事实卡、内容可用模块、FAQ、禁用表达和 Prompt 输入包。
- 审计层：来源索引、证据分级、待确认区和版本更新机制。
- 完整性自检层：权威参考对齐、分析完整性评分、实体覆盖统计和补强动作。

## 质量重点

- 品牌自述与第三方证据分级。
- 无法确认的数据进入待确认区，不进入强证据区。
- 真实数据只有在可访问、可引用、可标注时间时进入强证据区。
- 每个事实保留来源、更新时间、可信等级和可用场景。
- 实体清单和事实卡不能互相替代；实体清单回答“有哪些实体”，事实卡回答“哪些事实可复用”。
- 客户案例和个人信息必须做隐私与授权检查。
- Word/PDF 默认做排版溢出检查，尤其防止表格和 URL 向右溢出。
- HTML 可视化报告默认带固定跟随的目录菜单，适合长报告阅读；视觉节奏参考 kami 的长文档排版，但保留本 skill 的白底交付要求。
