# Yao GEO Panorama Audit

## 使用场景

- 项目启动、季度复盘、竞品追赶、投放前评估，或需要先建立 GEO 基线。
- 用户要先从目标品牌官网抓取公开信息，交叉验证品牌实体、产品、页面、内容资产、竞品和外部信源，再输出站内与站外系统方案。
- 面向品牌负责人、增长负责人、GEO 顾问、内容负责人输出机会地图和 P0 优先级。

## 不适用场景

- 只诊断一个 URL 或页面结构；应改用页面诊断类 skill。
- 只写榜单、对比、科普、标题或旧文改造；应改用内容生产类 skill。
- 只需要第三方问答平台答案表现和趋势监测；应改用监测类 skill。
- 只设计 GEO 效果追踪、归因字段或数据看板；应改用 `yao-geo-tracking`。
- 已有诊断结论，只需要拆 30/60/90 天执行计划；应改用实施方案类 skill。

## 必要输入

- 品牌名、官网、产品线、行业、区域、目标用户和业务目标。
- 已知竞品、已有内容清单、核心页面、品牌资料、销售资料和 PR 材料。
- 种子问题、核心关键词、客户咨询记录、历史 SEO 词库、站内搜索数据。
- 可选：GSC、GA、百度统计、CMS、CRM 线索、第三方舆情和媒体稿。

## 语言与命名约束

- Skill 名称必须遵守 `yao-geo-xxx` 规范；本 skill 固定为 `yao-geo-panorama-audit`。
- 默认输出语言为中文简体；报告正文、表头、字段名、状态值、判断标签和模块名称都应使用中文简体。
- 保留品牌名、产品名、平台名、URL、专有名词和用户英文素材，避免误译。
- 如用户明确要求双语或英文版本，先完成中文简体主版，再生成对应语言版本。

## 执行流程

1. 先按 `research-foundation.md` 建立诊断口径：GEO 全景诊断从官网事实源出发，以官网抓取、断言级核验、公开信源交叉验证和站内/站外机会地图为主线。
2. 抓取并归档目标品牌官网和官方入口：主页、产品页、方案页、价格页、案例页、文档页、帮助中心、博客、新闻稿、关于页、联系页和 sitemap。
3. 建立品牌实体档案，统一品牌名、别名、产品、服务、资质、客户、案例、价格、渠道、地域、适用场景和不适用边界。
4. 对官网断言做交叉验证：优先用官方二级来源、帮助文档、投资者资料、公开客户案例、媒体报道、百科、社区、行业报告和已授权资料核验；无法核验的内容标为待确认。
5. 生成用户问题覆盖矩阵，覆盖推荐、比较、替代、教程、价格、风险、真实性、购买决策和场景解决，用来检查官网是否能回答这些问题。
6. 补全 12 个诊断域，详见 `official-site-evidence.md`。
7. 按统一 GEO 特征模型评分：语义密度、结构规范性、可引用性、权威信号、可读性、鲁棒性、新颖性、跨域贡献，并增加完整性评分。
8. 输出机会地图，必须拆成站内方案和站外方案：站内包括页面重构、事实卡、FAQ、结构化摘要、内链、Schema、内容补齐；站外包括媒体、百科、公众号/视频号、社区、行业报告、合作伙伴、案例证据和第三方背书。

## 输出契约

- `GEO 全景诊断报告`
- `方法依据与评分口径`
- `权威参考映射`
- `官网抓取与事实交叉验证表`
- `官网抓取覆盖、实体档案、产品服务与商业信息地图`
- `用户问题覆盖矩阵`
- `技术可抓取与结构化数据诊断`
- `GEO 机会地图与优先级矩阵`
- `站内系统方案`
- `站外证据建设方案`
- `风险、假设与待确认项`
- `页面与内容资产修复清单`
- 默认四格式交付：Word（`.docx`）、PDF（`.pdf`）、HTML（`.html` 或完整 HTML 包）、Markdown（`.md`）。

## 校验清单

- 生成示例或正式报告时，必须真实产出 Word、PDF、HTML、Markdown 四种格式，并共用同一内容结构。
- 四格式生成后，必须按 `references/artifact-layout.md` 和 `references/quality-gates.md` 自 review；发现排版溢出、边框错位、行距异常、表格不可读或字段英文化时，先自行修复再交付。
- Word 表格必须显式控制总宽、网格列宽和单元格宽度；密集表需要拆成窄表，不能向右溢出或贴边。
- PDF 必须从已校验 HTML 渲染，检查页面尺寸、分页、表格边框和是否存在浏览器默认页脚。
- HTML 必须包含固定跟随的章节菜单栏，滚动时不遮挡内容。
- 不把 SEO 排名、官网流量或品牌知名度直接等同于 GEO 优先级。
- 不把“官网有页面”直接等同于“AI 可准确引用”；必须检查页面是否有可抽取的定义、表格、FAQ、来源、更新时间和断言级事实。
- 不把引用链接数量直接等同于可信度；必须检查引用是否真实支持对应断言。
- 品牌事实、客户案例、数据、资质和价格必须有来源；无法确认则标为待确认。
- 结论必须区分事实问题、结构问题、信源问题和执行问题。
- 机会地图必须有资源约束、P0/P1 优先级、负责人建议和验收口径。
- 报告必须系统、详细、完整：闭环、断言级证据、覆盖 12 个诊断域。
