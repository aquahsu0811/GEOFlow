# 交付物设计画像

## 交付物定位

GEO 全景诊断报告是战略诊断文档，不是内容文章，也不是后台看板。它需要帮助品牌负责人和增长负责人快速判断：官网事实源是否清楚、公开证据是否足够、站内与站外下一步资源先投向哪里。

## 阅读层级

1. 一页执行摘要：结论、P0、风险。
2. 一组方法依据：权威参考映射、官网抓取口径、评分模型。
3. 四类证据表：官网抓取覆盖、断言交叉验证、来源台账、风险与待确认项。
4. 六类诊断：品牌实体、产品商业、内容问题、技术结构、外部信源、竞品差距。
5. 两类方案：站内系统方案和站外证据建设方案。

## 版式方向

- 克制、可信、可复制到项目管理工具。
- 按 `kami` 的专业文档纪律控制层级、留白、表格节奏和阅读密度，但本报告固定使用白底背景。
- 表格优先，但每个表格要有判断句。
- 不做营销海报式视觉，不使用装饰性背景。
- HTML/PDF/Word/Markdown 保持同一章节结构，且都以中文简体为默认输出。
- HTML、PDF、Word 要重点检查对齐、边框、列宽、分页、行距和长文本换行，不能出现内容溢出。
- HTML 需要固定跟随的章节菜单，方便阅读长报告；菜单用白底、细边框、紧凑文字链接，不做浮夸视觉，不遮挡正文。
- 示例报告生成后必须自 review 并自行修复，再对外提供文件地址。

## 容易显得低质的信号

- 标题空泛，如“综合分析”“优化建议”。
- 表格只有结论没有来源。
- 大量英文表头或状态值。
- 所有建议都写成“加强内容建设”。
- 没有官网抓取范围、核验日期和来源台账。
