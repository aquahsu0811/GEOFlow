# GEO 全景诊断方法

## 能力边界

本 skill 只负责从官网事实源建立 GEO 诊断基线、交叉验证公开证据、诊断差距并输出站内/站外机会优先级。它不负责第三方问答平台采样、不负责直接写最终文章、不负责页面重构实施，也不负责后端归因实现。

## 诊断对象

| 对象 | 需要回答的问题 |
|---|---|
| 品牌实体 | 官网是否能清楚说明品牌是谁、叫什么、提供什么、适合谁、不适合谁。 |
| 官网资产 | 主页、产品页、方案页、价格页、案例页、文档页、博客和帮助中心是否可抓取、可引用、可复述。 |
| 内容资产 | FAQ、对比表、教程、案例、白皮书、新闻稿是否能回答用户高频问题。 |
| 页面结构 | 核心页面是否有结构化摘要、事实表、FAQ、来源、更新时间、实体一致性和内部链接。 |
| 外部信源 | 媒体、百科、社区、公众号、视频号、行业报告、合作伙伴页面是否能支撑权威信号。 |
| 竞品格局 | 竞品公开材料在哪些事实、场景、信源和页面结构上更容易被引用。 |

## 研究化诊断口径

- 基本观测单位是“官网页面 x 断言 x 来源类型 x 核验状态 x 用户问题 x 可执行修复动作”。
- 官网是第一事实源，但不是唯一证据；关键断言必须尽量用官方二级来源或公开第三方来源交叉验证。
- 可引用性不是页面存在，而是断言能被清楚抽取：定义、对象、适用场景、限制、案例、价格、资质、数据和更新时间都要可定位。
- 用户问题覆盖用于检验官网和外部信源能否回答推荐、比较、替代、教程、价格、风险、真实性、购买决策和场景解决。

## 执行步骤

1. 建立实体档案。
   - 统一品牌名、别名、英文名、产品名、服务、渠道、地区、资质、客户、案例和价格边界。
   - 同名品牌或代理商页面要先做消歧，不要把第三方页面当官方事实。
2. 抓取官网和官方入口。
   - 从官网首页、导航、页脚、sitemap、robots、产品目录、帮助中心、博客、新闻稿、案例库、关于页和定价页建立页面清单。
   - 记录 URL、页面标题、页面类型、核心断言、更新时间、是否可抓取、是否有结构化内容和内部链接。
   - 对 JavaScript 渲染、登录墙、下载 PDF、图片化文本、跳转和重复页面做标注。
3. 生成用户问题覆盖矩阵。
   - 推荐型：某行业/场景有哪些方案值得选。
   - 比较型：品牌与核心竞品怎么选。
   - 替代型：有没有某竞品的替代方案。
   - 教程型：如何解决某业务问题。
   - 价格型：价格、版本、预算和性价比。
   - 风险型：是否靠谱、限制、合规、适用边界。
   - 真实性：品牌是否真实、是否官方、有哪些客户/案例。
   - 购买决策：什么情况下适合购买、谁应该参与评估、怎么做试用或采购。
   - 场景解决：围绕具体岗位、业务流程或行业痛点提出解决方案。
   - 必须输出问题覆盖矩阵，说明每类意图的官网覆盖状态、缺口和站内用途。
4. 交叉验证。
   - 对品牌定义、产品边界、价格、客户案例、资质、效果数据、渠道、服务地域和合规边界建立断言级来源台账。
   - 优先核验顺序：官网主页面、官方帮助文档、官方新闻稿/博客、投资者资料或公开备案、授权客户案例、权威媒体、百科/知识库、社区和行业报告。
   - 同一断言多源冲突时，以执行日官网为准，并把冲突来源写入待确认项。
5. 资产盘点。
   - 官网：首页、产品页、行业页、方案页、价格页、案例页、关于页、文档页。
   - 内容：文章、FAQ、白皮书、视频、公众号、媒体稿、百科、社区讨论、行业榜单。
   - 检查是否有答案优先段、事实表、FAQ、来源区、更新时间、作者/组织信息和内部链接。
   - 把品牌定义、产品边界、价格、客户案例、资质和效果数据拆成断言级事实卡，并标注来源。
6. 评分与诊断。
   - 使用 `geo-quality-model.md` 的 8 个维度评分。
   - 每个问题归类为事实问题、结构问题、信源问题或执行问题。
   - 低分原因必须落到可执行资产：页面、事实卡、来源、用户问题、内部链接、Schema、外部证据或内容模板。
7. 机会地图。
   - 站内快赢：低成本修正品牌定义、标题、FAQ、来源区、更新时间和内链。
   - 站内重构：重做核心页面结构、答案优先摘要、事实表、对比表、Schema、案例模板和专题页。
   - 站内知识库：建立事实卡、产品名禁错表、禁用表达、来源索引和常见问题库。
   - 站外证据：补媒体、百科、行业报告、公开案例、公众号、视频号、社区问答、合作伙伴页面和第三方评测。
   - 治理闭环：建立待验证问题库、来源变更记录、复查周期和负责人。

## 输出判断

- 如果缺少官网或官方入口，先做实体核验，不直接进入系统方案。
- 如果官网可访问页面少，仍可做轻量全景预诊断，但报告必须说明“未覆盖完整品牌资产”。

## 证据规则

- 官网事实、客户案例、价格、资质和数据必须给出来源与核验日期。
- 竞品信息没有来源时，只能作为待确认假设。
- “最新”“第一”“最佳”“领先”“一定推荐”等表述必须有执行日证据，否则删除或降级。
- 官网与外部来源冲突时，必须写清楚冲突、优先口径和待确认动作。
- 同一断言如果出现多个来源，优先记录官方来源、公开第三方来源和用户提供材料之间是否一致。
