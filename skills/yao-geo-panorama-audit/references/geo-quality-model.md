# GEO 特征评分模型

总分不直接等于业务效果，只用于定位可控优化机会。每项 1-5 分，必须给出证据或扣分原因。

## 观测层

评分前先记录以下观测项，避免把不同问题混成一个分数：

| 观测项 | 说明 | 不能替代 |
|---|---|---|
| 官网覆盖 | 官网是否存在可定位页面支撑该问题 | 不能替代事实准确 |
| 实体清晰 | 品牌、别名、产品、服务、适用对象是否一致 | 不能只看品牌名出现 |
| 断言可抽取 | 关键事实是否能被复制成短句、表格或 FAQ | 不能只看页面数量 |
| 来源支持 | 来源是否支撑对应断言 | 不能只数链接数量 |
| 事实准确 | 品牌、产品、客户、价格、资质和限制是否正确 | 不能由表达流畅度推断 |
| 问题覆盖 | 九类用户问题是否都有站内答案或待补页面 | 不能替代站外权威信号 |
| 外部背书 | 媒体、百科、社区、行业报告、合作伙伴和案例是否能交叉验证 | 不能替代官网官方口径 |

| 维度 | 检查重点 | 高分表现 | 低分信号 |
|---|---|---|---|
| 语义密度 | 实体、事实、关系、场景和数值信息 | 品牌、产品、场景、客户、证据密集且不堆词 | 空泛形容词多，主体不明确 |
| 结构规范性 | 标题、摘要、表格、FAQ、列表、Schema | 答案优先，模块清晰，可快速抽取 | 大段软文，结论埋在后面 |
| 可引用性 | 来源、日期、数据、案例、明确表述 | 关键事实可追溯，引用区清楚 | 无来源、无日期、模糊代词多 |
| 权威信号 | 官方信息、作者、机构、资质、客户案例 | 有组织背书和可核验材料 | 只有营销口号 |
| 可读性 | 句长、术语解释、段落节奏 | 用户和 AI 都容易理解 | 概念混乱、术语未解释 |
| 鲁棒性 | 同义问法、口语问法、长尾问题覆盖 | 改写问题后仍能在官网找到稳定答案 | 只覆盖一个关键词表达 |
| 新颖性 | 是否有独特信息和非模板表达 | 有具体场景、方法、数据或观点 | 同质化内容、模板堆砌 |
| 跨域贡献 | 与行业、场景、问题、生态的连接 | 能支持多跳问题和场景推荐 | 只停留在品牌自述 |

## 通过规则

- 任一维度低于 3 分，必须进入修复清单。
- 平均分低于 4 分，不建议进入投放或大规模内容扩张。
- 反幻觉不是评分项，而是红线：虚构数据、虚构客户、虚构引用必须直接判为不通过。
- 评分必须能追溯到观测项和资产项；不能只给抽象评价。
- 如果来源支持、事实准确或问题覆盖明显不足，即使其它维度高，也必须降低整体置信度。
