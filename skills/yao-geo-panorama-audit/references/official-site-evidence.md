# 官网抓取与证据核验规范

## 抓取范围

| 页面类型 | 必查内容 | 主要用途 |
|---|---|---|
| 首页 | 品牌定义、主张、核心产品、目标用户、入口链接 | 建立品牌实体和导航优先级 |
| 产品页 | 产品名、模块、功能边界、适用对象、截图或流程 | 校准产品结构和功能断言 |
| 方案页 | 行业、场景、岗位、痛点、解决路径 | 支撑场景解决和推荐型问题 |
| 价格页 | 套餐、价格口径、限制条件、咨询入口、更新时间 | 降低价格幻觉和过期信息风险 |
| 案例页 | 客户名称、行业、问题、方案、结果、授权状态 | 支撑权威信号和外部传播 |
| 帮助中心 | 教程、配置步骤、限制、集成说明、故障处理 | 支撑教程型和风险型问题 |
| 博客/新闻稿 | 观点、方法、产品更新、行业解释 | 支撑新颖性和时效性 |
| 关于页 | 公司主体、历史、团队、资质、合规、联系方式 | 支撑真实性和消歧 |
| sitemap/robots | 页面发现、可抓取性、禁止抓取边界 | 判断资产覆盖和抓取风险 |

## 统一字段

| 字段 | 说明 |
|---|---|
| `source_id` | 来源编号，便于引用和回查。 |
| `url` | 页面 URL 或官方文件入口。 |
| `source_type` | 官网主页、产品页、帮助文档、案例页、博客、新闻稿、投资者资料、第三方媒体、百科、社区、行业报告等。 |
| `claim` | 从页面抽取的断言，必须是可核验的短句。 |
| `claim_type` | 品牌定义、产品边界、价格、客户案例、资质、效果数据、渠道、地域、限制、合规、场景、竞品比较。 |
| `page_title` | 页面标题或文件标题。 |
| `published_or_updated_at` | 页面公开日期或更新时间；无法确认则写“未标注”。 |
| `crawl_time` | 本次抓取或核验的绝对日期。 |
| `crawlability` | 可抓取、需 JS 渲染、PDF、图片化文本、登录墙、跳转、不可访问。 |
| `cross_check_source` | 用来交叉验证的官方二级来源或公开第三方来源。 |
| `verification_status` | 已核验、部分核验、来源冲突、待确认、不可核验。 |
| `geo_use` | 可用于品牌实体、价格说明、场景 FAQ、教程、案例、外部证据、竞品对比等。 |
| `risk_note` | 过期、模糊、夸大、缺少授权、与其它来源冲突、无法直接引用等风险。 |
| `action` | 站内修复或站外建设动作。 |

## 核验顺序

1. 优先使用目标品牌官网和官方入口。
2. 同一断言优先找官方二级来源交叉验证，例如帮助中心、产品文档、新闻稿、价格页、投资者资料、公开备案或授权案例。
3. 官方资料不足时，使用可信公开第三方来源，例如权威媒体、行业报告、百科、应用市场、合作伙伴页面、社区问答。
4. 第三方来源只用于验证和补充，不能覆盖官网的最新官方口径；发现冲突时写入待确认。
5. 销售资料、CRM 记录、客户访谈和内部材料只能作为内部辅证；没有公开授权时不得写成公开事实。

## 站内/站外拆分

| 方案类型 | 典型动作 | 验收口径 |
|---|---|---|
| 站内品牌实体 | 品牌定义页、别名表、产品边界、适用/不适用场景 | 用户问题能在官网找到统一官方答案 |
| 站内页面结构 | 答案优先摘要、事实表、FAQ、对比表、来源区、更新时间 | 核心页面可被复制、引用和快速理解 |
| 站内内容补齐 | 推荐、比较、替代、教程、价格、风险、真实性、购买决策、场景解决专题 | 九类用户问题都有可定位页面 |
| 站内技术结构 | sitemap、内链、Schema、可抓取文本、PDF 文本化、图片替代文本 | 抓取与引用路径清晰 |
| 站外权威信号 | 媒体报道、百科、行业报告、合作伙伴页、公开客户案例 | 关键断言有官网外证据支撑 |
| 站外内容生态 | 公众号、视频号、社区问答、演示视频、评测文章 | 中文场景和长尾问题有公开信源 |

## 完整诊断域

正式报告不能只停留在“官网有什么页面”。至少检查以下诊断域，并说明未覆盖原因：

| 诊断域 | 必查问题 | 输出形式 |
|---|---|---|
| 官网抓取覆盖 | 是否覆盖导航、页脚、sitemap、核心页面、隐藏入口、PDF 和帮助中心。 | 官网抓取清单 |
| 品牌实体消歧 | 品牌名、别名、英文名、公司主体、官网、社媒、百科和同名品牌是否一致。 | 实体关系表 |
| 产品服务模型 | 产品线、模块、服务、套餐、对象、旧名/新名、功能边界是否清楚。 | 产品服务地图 |
| 价格商业口径 | 价格、版本、Seat、Credit、试用、咨询制、限制条件和更新时间是否明确。 | 商业信息表 |
| 客户案例证据 | 客户、行业、规模、问题、方案、结果、授权状态是否可公开。 | 案例证据矩阵 |
| 内容深度 | 推荐、比较、替代、教程、价格、风险、真实性、购买决策、场景解决是否有站内答案。 | 用户问题覆盖矩阵 |
| 技术可抓取性 | robots、sitemap、canonical、状态码、JS 渲染、PDF 文本化、图片 alt 是否合格。 | 技术可抓取表 |
| 结构化数据 | Organization、Product/Service、FAQ、Breadcrumb、Article、CaseStudy 或等价类型是否匹配页面可见内容。 | Schema 建议表 |
| 权威与信任 | About、作者/组织、来源、更新时间、资质、合规、联系方式和纠错机制是否充分。 | 信任信号表 |
| 竞品与替代 | 竞品在页面结构、证据类型、外部信源和场景覆盖上强在哪里。 | 对标差距表 |
| 站外证据生态 | 媒体、百科、公众号、视频号、社区、行业报告、合作伙伴和应用市场是否支撑关键断言。 | 站外证据地图 |
| 风险与假设 | 事实冲突、无法核验、合规边界、客户授权、过期数据和执行资源限制是否列明。 | 风险台账 |

## 交叉验证深度

- `一级核验`：官网主页面与页面内可见内容一致。
- `二级核验`：官网主页面与官方帮助文档、价格页、博客、新闻稿、客户故事或投资者资料一致。
- `三级核验`：官方事实与公开第三方来源一致，例如媒体、百科、行业报告、合作伙伴、应用市场和社区。
- `冲突处理`：同一断言有冲突时，报告必须写出“采用口径、冲突来源、待确认动作、负责人建议”。
- `完整性处理`：没有足够来源时，不允许直接补结论；必须列为站内补证或站外补证任务。

## 禁止项

- 不要求本 skill 采集第三方问答平台答案。
- 不得把官网页面数量写成 GEO 成功概率。
- 不得把内部销售材料、未授权客户名称或未公开效果数据写成公开事实。
- 不得只给“多发内容”这类泛建议，必须落到页面、断言、来源、负责人和验收口径。
