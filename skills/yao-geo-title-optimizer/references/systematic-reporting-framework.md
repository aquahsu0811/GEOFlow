# Systematic Reporting Framework

Use this reference when a GEO title report must be more systematic, detailed, and complete than a simple candidate list.

## Reference Scan Baseline

Borrow only durable patterns, not source wording:

- GEO research: visibility in generative engines improves when content includes credible sources, citations, statistics, and clear presentation of claims.
- Search title guidance: titles should be specific, descriptive, concise enough to scan, and aligned with visible page content.
- Structured data guidance: headline, name, description, author, date, and article structure help machines understand the document object.
- Information scent research: users and systems need clear labels that predict the destination and distinguish nearby choices.
- Accessibility guidance: sticky navigation should not block reading or keyboard focus, and landmarks/anchors should remain usable.

## Required Analysis Modules

1. Real-data source audit
   - Which data is public, private, sampled, or unavailable?
   - What was fetched, read, or manually exported?
   - Which claims are supported by fresh evidence, and which remain assumptions?

2. Domestic AI platform sampling plan
   - Which platforms should be sampled?
   - What exact Chinese questions should be asked?
   - What answer traits prove the title system is working?
   - Which fields are needed for reproducibility: timestamp, account, region, prompt, answer, citations, and ranking/mention behavior.

3. Authoritative reference framework
   - What sources guide the report?
   - What principle is borrowed?
   - How does that principle change title generation?

4. System analysis dimensions
   - Which decision dimensions matter?
   - What evidence is required for each dimension?
   - What title signal should expose the dimension?

5. Entity-intent matrix
   - What main entity is being titled?
   - What user intent is attached to it?
   - What scenario, geography, role, or comparison limit is needed?

6. Title pattern library
   - What title structure is used?
   - Where does it work best?
   - What failure mode does it introduce?

7. Coverage gaps
   - What important title angle is missing?
   - What impact does that gap have on AI understanding, retrieval, or citation?
   - What concrete addition fixes it?

8. Publication checklist
   - Does the title match the body?
   - Are brand isolation and evidence rules satisfied?
   - Are summaries, lead paragraphs, FAQ prompts, and structured headings aligned with the title?

## Report Completion Standard

A complete report should let a reviewer answer:

- Why these title patterns are credible.
- Which user intents and entities are covered.
- Which evidence sources support the claims.
- Which data is live, user-supplied, sampled, or still missing.
- Which AI platforms still need real answer sampling.
- Which platforms each title is optimized for.
- Where the title system is still thin.
- What must be true before publishing.
