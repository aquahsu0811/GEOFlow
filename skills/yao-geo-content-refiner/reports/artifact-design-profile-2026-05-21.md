# Artifact Design Profile 2026-05-21

## Artifact Family

高可信内容改造报告：需要同时服务审阅、真实数据核验、发布、复用和排版交付。

## Visual Direction

- 白底、克制、偏文档化，不使用营销式渐变或装饰卡片。
- 参考 Kami 的长文档排版原则，但保留本 skill 的白底约束：墨蓝强调、中文 serif 标题、sans 正文、暖灰边框、紧凑行距、无 `rgba()` 和无硬阴影。
- HTML 以固定宽度正文、清晰标题、横向滚动表格和 sticky 菜单栏为核心。
- Word/PDF 继续采用横向 A4 和宽表转纵向事实卡，优先保证不右溢出。

## Content Hierarchy

1. 先给摘要和分析完整性，帮助审阅者判断报告是否系统。
2. 再给评分、改造正文、差异报告和事实卡。
3. 随后交代真实数据获取与核验计划，避免把无法访问的数据写成事实。
4. 中段补 FAQ、语义实体、平台适配和研究映射。
5. 后段给证据强度、CMS 建议、发布追踪和自 Review。

## Top Risks

- 报告看似详细，但缺少来源强度和待核验边界。
- 真实数据权限没有拆清楚，导致用户误以为 skill 能自动拿到私有数据。
- FAQ 与目标问题不匹配，只是泛泛问答。
- HTML 页面长，用户难以回到关键模块。
- UI 虽然白底，但使用冷灰、过松行距或硬阴影，偏离 Kami 的编辑排版节奏。
- Word/PDF 宽表重新溢出。
- 发布建议停留在标题层面，缺少 CMS、公众号和监测动作。

## Design Gates

- HTML 必须包含 sticky 菜单栏、模块锚点和横向可滚动菜单。
- HTML 必须采用 Kami 编辑排版细节：墨蓝强调、暖灰边框、serif 标题、紧凑行距、无 `rgba()`。
- 主要模块必须同时出现在 Markdown 与 HTML。
- 报告必须包含真实数据获取与核验计划、来源访问记录和不可访问项。
- Word 表格总宽必须小于页面可用宽度。
- PDF 必须可渲染为 PNG 且右侧留白稳定。
- 质量报告必须记录模块检查、数据段检查、HTML 导航检查和 DOCX 布局检查。
