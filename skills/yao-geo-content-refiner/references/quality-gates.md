# 质量门

## 内容质量门

- 必须包含原文 GEO 评分、改造版文章、差异报告、原子事实卡、FAQ、理论映射、证据强度与缺口和 CMS HTML 建议。
- 必须包含分析完整性总览、真实数据获取与核验计划、语义与实体地图、平台适配矩阵、发布与追踪建议。
- 新增事实必须有来源或证据状态，弱证据不得写成强结论。
- 医疗、金融、法律等敏感领域必须保留限制条件。
- 参考依据必须覆盖 GEO、RAG/引用生成、原子事实核验、长上下文鲁棒性、自 Review 和可信度框架。
- 真实数据必须标注获取方式、权限边界、访问时间、新鲜度和失败处理；未授权数据不得访问或推测。

## Word/PDF 排版质量门

- 四格式文件必须真实存在且非空。
- Word 必须检查所有表格 `tblGrid` 总宽，任何表格都不得大于页面可用宽度。
- Word 中超过 4 列的表格必须转成纵向事实卡，不允许 6-8 列宽表直接输出。
- Word 长 URL 必须可换行，不得向右穿出单元格。
- PDF 必须渲染成 PNG，并检查右侧留白；不得出现文字或表格贴边、裁切、乱码。
- HTML 必须白底、表格可横向滚动、无本地绝对路径。
- HTML 必须有 sticky 菜单栏，菜单锚点覆盖主要报告模块，滚动时固定跟随且不遮挡正文。
- HTML 必须符合 kami 打印安全细节：无 `rgba()`，墨蓝强调、暖灰边框、serif 标题、紧凑行距。

## 失败处理

发现任一右溢出、贴边、裁切、表格过窄或长 URL 穿出页面时，必须先修渲染逻辑并重新生成 Word/PDF/HTML/Markdown，不能只手工改单个文件。

发现分析模块缺失、真实数据计划缺失、证据强度缺失、平台适配矩阵为空、HTML 菜单缺失时，同样必须先修 skill 逻辑和输入模型，再重新生成四格式报告。
