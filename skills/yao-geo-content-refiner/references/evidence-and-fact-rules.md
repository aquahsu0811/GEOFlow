# 证据与事实规则

## 证据状态

| 状态 | 含义 | 输出方式 |
|---|---|---|
| 已核验 | 来源能直接支持断言。 | 可作为正式结论。 |
| 用户提供 | 用户资料声明，但未公开核验。 | 可写，但标注来源类型。 |
| 待核验 | 有线索但未确认。 | 进入缺口清单，不写成强结论。 |
| 证据不足 | 没有来源或来源不支持。 | 删除、弱化或标注为待补证。 |

## 证据强度

| 强度 | 典型来源 | 使用方式 |
|---|---|---|
| A | 官方页面、官方公告、监管披露、标准文件、原始研究论文。 | 可支持核心事实，但仍需标注时间和适用范围。 |
| B | 用户授权资料、白皮书、产品手册、合同附件、可核验案例。 | 可支持业务场景，需标注“用户提供”或“资料版本”。 |
| C | 第三方媒体、行业报告、评测、公开评论。 | 只能作为背景或线索，不能替代官方事实。 |
| D | 无来源、过期页面、转述、截图无法追溯、销售口头描述。 | 不得写成强结论，进入缺口或删除。 |

## 断言级映射

每条关键断言都要能落到以下字段：事实项、来源、来源类型、证据强度、证据状态、备注。若一个段落包含多个事实，必须拆成多张原子事实卡；若来源只能支持部分事实，不能把整句标为“已核验”。

## 真实数据状态

| 状态 | 含义 | 输出方式 |
|---|---|---|
| 可公开核验 | 无需登录即可访问，来源能直接支持字段。 | 写入来源访问记录和证据强度表。 |
| 需用户资料 | 需要用户上传文件、截图、合同、销售资料或知识库内容。 | 标注“用户提供”，并列出资料缺口。 |
| 需授权连接 | 需要连接器、API、数据库、CRM、BI 或内部系统权限。 | 输出授权需求，不尝试未授权访问。 |
| 不可访问 | 受限、付费、登录、隐私或未授权数据。 | 不使用，不推测，进入待核验缺口。 |

## 事实红线

- 不编造客户案例、价格、资质、效果数据、排名、收益、疗效或合规结论。
- 新增来源必须真实存在，且能支持对应断言。
- 弱证据不得写成强结论。
- 医疗、金融、法律、保险、教育升学等内容必须保留限制条件，不承诺结果。
- 不把跨区域官网资料直接推导为中国大陆价格、合规、本地交付、微信生态集成或客户成功承诺。
