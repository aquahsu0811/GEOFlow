# 国内 AI 平台适配

改造后的内容应同时服务 DeepSeek、豆包、千问、Kimi、腾讯元宝和微信生态。

| 目标 | 改造要求 |
|---|---|
| 开头可直接回答 | 第一屏回答目标问题，给出结论、适用对象和限制条件。 |
| 段落可独立引用 | 每段只承载一个核心事实或判断。 |
| 事实可核验 | 数据、案例、价格、资质和时间都要有来源或证据状态。 |
| 结构可扫描 | 使用 H2/H3、短段、表格、步骤、FAQ 和来源列表。 |
| 问题可重写 | 输出同义问法、场景问法、反向问法和追问。 |
| 表格可抽取 | 表头稳定，字段中文，单元格只放单一含义。 |
| 微信生态可读 | 长句拆短，宽表可拆为列表或事实卡。 |

## 平台适配矩阵要求

每份报告必须输出平台适配矩阵，至少覆盖：

- 通用 AI 问答：开头直接回答、事实卡可引用、来源清晰。
- 深度研究型问答：保留证据强度、缺口、反向边界和更新时间。
- 对比/采购型问答：补适用对象、不适合对象、替代方案和采购前核验项。
- 微信生态阅读：减少长表，拆成短段、列表、FAQ 和事实卡。
- CMS/官网发布：保留 H1、摘要、目录、FAQ、来源和结构化表格。

## 查询重写要求

FAQ 不只输出“标准问答”，还要覆盖同义问法、反向问法、场景问法和追问。对于产品页或品牌页，至少包含：是什么、适合谁、解决什么问题、有什么限制、证据来自哪里、是否适合本地场景。
