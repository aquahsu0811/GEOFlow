# Authoritative Reference Framework

Use these references to make analysis and reports more systematic, detailed, and complete.

## Primary References

| Reference | Authority | What To Borrow |
| --- | --- | --- |
| Schema.org `Organization`, `Product`, `Brand`, `FAQPage`, `Article` | Schema.org vocabulary | Entity typing, entity properties, product/service modeling, FAQ module structure |
| W3C PROV-O | W3C Recommendation | Provenance model: entity, activity, agent, source lineage, generated/derived facts |
| W3C SKOS | W3C Recommendation | Controlled vocabulary, preferred labels, aliases, broader/narrower/related concepts |
| Google Search Central structured data gallery | Google Search Central | Search-oriented page modules and structured-data categories |
| Google Search Central FAQ structured data | Google Search Central | FAQ/Q&A distinction and authority-sensitive FAQ use |
| Local `ai-brand-kb-builder` | Local skill reference | Formal AI brand knowledge-base section stack and fact-dense writing style |

## How To Apply

- Use Schema.org-style entity classes to avoid an incomplete entity inventory.
- Use PROV-O thinking to separate source, evidence, extraction activity, generated fact, and report output.
- Use SKOS thinking to capture preferred names, aliases, query terms, broader/narrower concepts, and related concepts.
- Use Google structured-data thinking to keep report modules extractable by search and AI systems.
- Use `ai-brand-kb-builder` to keep the final report readable as a formal knowledge base, not only as an audit ledger.

## Minimum Reference-Derived Dimensions

Every analysis should check these dimensions:

1. Identity: organization, brand, legal/company names, website, regions.
2. Offerings: products, services, features, plans, prices, packages.
3. Capabilities: technology, AI features, workflows, integrations, support boundaries.
4. Evidence and provenance: source, publisher, date, extraction note, confidence.
5. Controlled vocabulary: preferred label, aliases, Chinese/English names, query terms.
6. Metrics: numbers, date/scope, denominator, source.
7. Cases and social proof: customers, case status, aggregate proof, privacy boundary.
8. Timeline: launch, update, milestone, financial/customer metric.
9. Competitive context: competitors, comparison dimensions, non-ranking boundary.
10. Governance: prohibited expressions, pending claims, update cadence, review owner.
11. Domestic AI adaptation: Kimi, Qianwen, DeepSeek, Doubao, Yuanbao reuse notes.
12. Data access and freshness: public/user-provided/authenticated/manual/unavailable acquisition mode, freshness cadence, and strong-evidence eligibility.
13. Output usability: Markdown/HTML/Word/PDF readability, sticky HTML menu, source auditability.
