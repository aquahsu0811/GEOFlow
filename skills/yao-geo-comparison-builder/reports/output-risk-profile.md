# 输出风险画像

- 事实风险：真实品牌价格、客户数量、集成数量和功能范围可能变化，必须以访问日来源为准。
- 合规风险：CRM 选型可能涉及数据合规、跨境传输、采购条款和行业监管，报告只能提出核验项。
- 公平风险：目标品牌内容容易写成单向推荐，必须保留竞品适用场景。
- 格式风险：对比内容天然表格多，必须拆表并做 DOCX 后处理以保障 Word/PDF/HTML 可读。
- 完整性风险：只写核心能力会导致报告像营销文案，必须补充维度模型、来源质量、风险治理和落地清单。
- 导航风险：HTML 长报告如果没有固定目录，读者难以定位章节；固定目录必须避免遮挡标题和打印污染。
- 数据访问风险：公开网页可能变化或访问失败，私有系统需要用户授权；报告必须记录访问模式、访问结果和降级结论。
- 版式一致性风险：Kami 风格要求暖米纸底、暖灰、油墨蓝和紧凑行距；若混入冷灰、纯白大面积背景、rgba tag 或 1.6+ 行距，会降低 PDF/HTML 的专业感。
