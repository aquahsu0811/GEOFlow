# 真实数据获取与验证

## 数据获取模式

- 公共网页模式：使用公开官网、价格页、产品目录、帮助中心、新闻稿、PDF、公开案例和公开文档。必须记录 URL、访问日期、访问方式、HTTP 状态或等价可访问证据。
- 用户文件模式：使用用户上传的合同摘录、报价单、销售资料、知识库、案例、竞品清单或内部备注。必须标注“用户提供资料”，不可伪装成公开来源。
- 授权连接器/API 模式：只读取用户已授权范围内的数据，例如 CRM、飞书、Google Drive、数据库或内部 API。必须记录连接器名称、权限范围、读取对象和脱敏规则。
- AI 平台采样模式：仅在用户要求采样 DeepSeek、豆包、千问、Kimi、腾讯元宝等平台回答时启用；需要单独记录采样问题、时间、平台、账号/地区约束和输出快照。
- 离线假设模式：当无法联网或用户未提供资料时，只能使用用户输入和显式假设，不能输出真实品牌事实。

## 权限边界

- 不读取未授权后台、登录后页面、付费报告、内部客户名单、合同全文、个人数据或敏感经营数据。
- 不绕过登录、验证码、robots 约束、付费墙或访问限制。
- 私有资料必须由用户提供或通过用户授权连接器读取，并在报告中标注来源类型。
- 动态数据如价格、版本、套餐、Credits、add-ons、客户数量、案例名单和集成数量必须写访问日期和有效性边界。

## 来源验证字段

每条来源至少记录：

- 来源 ID。
- 来源名称。
- URL 或文件名。
- 来源类型：官方页面、官方价格/目录、帮助中心、新闻稿、PDF、用户资料、授权系统、方案类型假设。
- 访问方式：web、browser、API、connector、uploaded-file、user-provided、assumption。
- 访问日期。
- HTTP 状态或读取状态。
- 支撑事实。
- 置信边界。
- 动态性：低、中、高。

## 数据缺口处理

- 公开来源不存在或访问失败：保留来源 ID 的访问失败记录，但不把该来源支撑为事实；结论降级为“需核验”。
- 竞品资料不足：删除该维度或把它转为核验项，不输出后台提示。
- 私有数据未授权：明确写成“需要用户提供或授权后核验”，不得猜测。
- 来源冲突：优先官方来源、合同/报价单、用户授权系统，再用第三方研究；冲突必须进入风险与治理地图。

## 输出要求

- 正式报告必须包含“真实数据获取说明”或等价模块。
- 启用公共网页、API 或连接器时，必须输出 `source-verification.json`。
- `quality-report.json` 必须记录 `real_data_access_check`：访问模式、可核验来源数量、失败来源数量、是否存在未授权数据、是否存在降级结论。
