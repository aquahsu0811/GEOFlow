# Examples

`yao-geoflow-cli` does not commit runnable public job examples by default because the skill operates against an already deployed GEOFlow system with local credentials and environment-specific IDs.

Safe public examples for this skill should only include:

- sanitized command intent
- non-sensitive placeholder IDs
- no tokens, no passwords, and no private URLs
