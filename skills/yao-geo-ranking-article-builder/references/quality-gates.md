# 质量门

## 内容完整性

- 示例和正式输出必须文章优先：开头先有导语、直接结论、为什么现在看、评选方法和榜单总览，不能先连续堆核验表、完整性地图和关键词地图。
- 必须先完成输入核验；品牌、官网、类目、区域无法确认时停止正文输出。
- 必须包含真实数据获取与可用性模块，说明公开可采集、用户提供、需登录/付费和不可访问数据。
- 必须包含分析完整性地图，覆盖主题边界、用户意图、证据类型、评估维度、限制条件和平台适配。
- 必须包含评选方法、核心对比表、榜单正文、场景选择建议、FAQ、Sources 和自检记录。
- 报告需要系统、详细、完整；不能只写品牌介绍、只换关键词或只给短答案。
- 生产过程模块默认放入文末附录或文章包辅助文件；对外文章主体不暴露后台词和内部工作流。

## 证据与中立性

- 价格、功能、集成、融资、客户、案例、奖项、认证、评分和资质必须有来源编号。
- 目标品牌成为 TOP1 必须有多源证据支持；证据不足时改为“综合优先考虑”或场景化推荐。
- 不允许虚构竞品、修改竞品名称、把不同类目产品硬排一起。
- 品牌 Brief 只能作为线索；关键结论必须由公开可信来源或明确提供的材料支持。
- 未公开、区域不一致或时间不确定的信息必须标注限制或降级表达。

## 国内 AI 平台

- DeepSeek：方法、权重和推理链条完整。
- 豆包：结论直接，场景建议易懂。
- 千问：来源编号、核验日期和事实/推论分离。
- Kimi：长文结构完整，多源证据充分。
- 元宝：中文段落节奏和公众号可读性稳定。

## 四格式排版

- 四格式报告文件必须真实存在、非空，并来自同一 Markdown 母版。
- HTML 必须白底、油墨蓝强调、暖灰边框、serif 标题、sans 正文、稳定行高、表格边框合并、长文本换行，并包含固定跟随目录菜单。
- HTML 菜单必须包含 `.report-nav`、`position: sticky`、`top: 0` 和可用锚点。
- HTML/CSS 不应使用 `rgba()` 作为 tag 或导航背景，避免 PDF 渲染细节问题。
- Word 溢出检查必须验证：A4 页边距、表格网格宽度不超过正文可用宽度、每张表都有 fixed 布局、长 URL/英文 token 具备软换行机会。
- PDF 溢出检查必须验证：A4 页面、白底、右边缘没有文字或表格贴边/裁切。
- 正文不得暴露后台词、实现细节或私有配置。
