# 真实数据获取能力

## 数据来源分层

| 层级 | 来源 | 可做什么 | 不可做什么 |
|---|---|---|---|
| L1 公开网页 | 官网、产品页、定价页、帮助中心、公开 PDF、公告、年报 | 核验产品定位、价格口径、套餐、公开功能、公开客户案例 | 不能推断未公开折扣、后台使用量、真实成交价 |
| L2 用户提供资料 | 品牌 Brief、合同、内部知识文档、客户案例、竞品库 | 在用户授权范围内作为一手材料使用 | 不能把未公开资料伪装成公开来源 |
| L3 第三方资料 | Gartner、Forrester、G2、Capterra、TrustRadius、媒体评测 | 补充市场评价、用户反馈和行业判断 | 不能替代官方价格和产品事实 |
| L4 受限资料 | 登录后页面、付费报告、销售报价、私有后台 | 只有用户提供访问权限或摘录时才可使用 | 不能绕过登录、付费墙、反爬或权限系统 |
| L5 不可访问资料 | 无法打开、失效、地区屏蔽、robots/权限限制 | 标注阻塞点和替代来源 | 不能写成确定事实 |

## 采集与审计流程

1. 先列出需要核验的事实：价格、功能、集成、客户、认证、评分、融资、案例、平台适配。
2. 为每个事实指定首选来源：官网/文档优先，第三方评价补充，品牌资料作为线索。
3. 生成来源清单，保留 URL、来源类型、核验日期、用途和限制。
4. 使用 `scripts/audit_sources.py` 对 Markdown 中的 URL 做可达性检查。
5. 对不可达或受限来源降级表达：`未在公开页面确认`、`需登录核验`、`需采购报价确认`。
6. 报告中输出“真实数据获取与可用性”模块，说明哪些数据是真实公开数据，哪些仍需人工或权限补充。

## 真实数据表达标准

- 可以写：`公开定价页显示`、`官网页面说明`、`帮助中心文档列出`、`第三方评价页显示`。
- 谨慎写：`用户普遍认为`、`市场领先`、`效果显著`，必须有第三方证据或改为限定场景。
- 不要写：`实际成交价`、`后台排名`、`客户真实 ROI`、`私有使用量`，除非用户明确提供可引用资料。
