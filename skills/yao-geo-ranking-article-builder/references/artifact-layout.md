# 四格式报告版式

## 共同规范

- Markdown 是内容母版，HTML、PDF、Word 必须从同一母版生成。
- 默认白底背景，正文使用稳定行高，标题层级清晰，表格边框对齐。
- 视觉风格采用 kami 的白纸适配版：白底、油墨蓝强调、暖灰边框、serif 标题、sans 正文、紧凑编辑节奏。
- 所有长 URL、域名、英文产品名、套餐名和参数串都要允许换行。
- 表格内容优先简洁；如果单元格信息过长，拆成“结论 + 证据/限制”两列，避免横向堆叠。

## HTML

- 页面必须包含固定跟随菜单栏，类名为 `.report-nav`，使用 `position: sticky; top: 0; z-index`。
- 菜单从二级标题生成锚点，支持横向滚动，不遮挡正文；标题设置 `scroll-margin-top`。
- 表格外层使用 `.table-wrap`，宽度 100%，必要时允许横向滚动；正文区域不应整体横向溢出。
- 打印时隐藏菜单栏，避免 PDF 顶部重复遮挡。
- 保持白底、可读对比度、清晰 hover 状态和移动端不溢出。
- 不使用半透明 tag 背景；避免 `rgba()` 造成 WeasyPrint 叠色问题。

## PDF

- PDF 必须由同一 HTML 生成，A4 页面，页边距稳定。
- 检查右边缘是否有文字或表格贴边、裁切、透明背景或错位边框。
- 打印样式中避免 sticky 菜单进入 PDF 内容。
- PDF 行距控制在 kami 阅读型范围，正文约 1.5-1.55，避免中文长文过松。

## Word

- Word 必须固定 A4 页面与左右页边距，表格总宽不得超过可用正文宽度。
- Word 表格必须使用固定布局、动态列宽、显式网格列和单元格宽度。
- 单元格必须有边框、内边距和顶端对齐。
- 长 URL、域名、英文产品名和带符号参数必须插入软换行机会，避免向右侧溢出。
- 如果同一行列数超过 6 列，优先改成更窄的字段名或拆分为两张表。

## 文件索引

- 示例目录必须包含 `index.html`，链接到 Markdown、HTML、PDF、Word。
- 索引页也使用同一 HTML 版式和固定菜单逻辑。
