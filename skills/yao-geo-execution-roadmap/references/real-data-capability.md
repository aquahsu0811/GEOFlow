# Real Data Capability

## 能力边界

本 skill 可以基于真实数据生成实施方案，但不能默认绕过外部平台权限直接取得所有真实数据。真实数据能力分为四类：

| 模式 | 可做什么 | 不能做什么 |
|---|---|---|
| 用户提供数据 | 读取诊断、采样表、CRM、来源台账、页面清单 | 判断未提供数据为真实事实 |
| 公开网页核验 | 在允许联网时核验官网、官方文档、投资者关系、监管披露、可信第三方来源 | 把过期网页或无来源说法当成事实 |
| 平台采样导入 | 设计 DeepSeek、豆包、千问、Kimi、元宝采样协议，并读取用户导出的采样表 | 无授权访问用户账号或承诺平台实时引用 |
| 缺口治理 | 标出待补证、待授权、禁用、过期和需复测数据 | 用猜测填补商业、价格、客户案例或效果数据 |

## 报告必须输出

- 数据模式：说明本次报告使用的是用户提供、公开网页核验、平台采样导入，还是混合模式。
- 真实数据能力边界：说明可以取得哪些真实数据，哪些需要用户授权或外部采样。
- 真实数据采集计划：每项数据写清获取方式、来源要求、新鲜度规则、责任角色和验收口径。
- 来源新鲜度：来源台账必须写来源类型、证据等级、核验日期和新鲜度。
- 待补证缺口：不能确认的数据不得进入事实正文，只能进入补证或禁用流程。

## 新鲜度规则

| 数据类型 | 默认新鲜度 |
|---|---|
| 官网产品口径 | 30 天复核 |
| AI 产品、价格、套餐、API、平台入口 | 14 天复核 |
| 投资者关系、财务、客户规模 | 季度复核，必须带截至日期 |
| 国内 AI 平台答案采样 | 7-14 天复测 |
| 客户案例、效果数据、行业数据 | 发布前复核授权和统计口径 |

## 采集优先级

1. P0：官方事实、产品线、价格/套餐、平台采样基线、合规风险。
2. P1：第三方证据、竞品对比、客户案例、行业信源。
3. P2：低意图科普、边缘场景、历史资料和低置信外部评价。
