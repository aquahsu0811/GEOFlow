# Analysis Completeness Framework

## 目标

让 GEO 综合实施方案先系统分析，再生成计划。报告不能只出现任务清单，必须说明为什么做、缺什么、谁负责、如何验收、如何复测、如何治理。

## 十维分析矩阵

| 维度 | 必答问题 | 输出要求 |
|---|---|---|
| 业务目标 | 项目服务线索、成交、曝光、权威、招商还是产品教育？ | 目标、优先级、约束 |
| 问题覆盖 | 用户会如何问推荐、比较、替代、价格、实施、风险和合规？ | 问题簇、意图、优先级 |
| 品牌实体 | 品牌名、别名、产品线、资质、客户、案例是否清楚？ | 品牌事实卡 |
| 页面技术 | 页面是否可抓取、可理解、可结构化、可稳定引用？ | 页面改造清单 |
| 内容资产 | 是否覆盖高频问题、购买场景和对比场景？ | 内容矩阵 |
| 标题体系 | 标题是否匹配自然问法和 AI 摘要抽取？ | H1/H2/摘要句规范 |
| 知识库 | 事实是否可复用、可更新、可追溯？ | 事实底座、术语表、FAQ |
| 外部证据 | 第三方来源是否足以支撑关键断言？ | 来源台账、证据成熟度 |
| 平台适配 | DeepSeek、豆包、千问、Kimi、元宝动作是否不同？ | 平台动作表 |
| 监测治理 | 是否有基线、复测、风险、预算和决策机制？ | 监测计划、治理节奏 |

## 证据成熟度

| 等级 | 含义 | 处理 |
|---|---|---|
| A 官方已证实 | 官网、公告、年报、官方文档可直接证明 | 可写入核心事实页 |
| B 第三方可佐证 | 媒体、行业报告、合作伙伴、公开评测可佐证 | 可用于外部证据 |
| C 内部待授权 | 销售数据、客户案例、项目成效需要授权 | 先入待审核区 |
| D 需要补证 | 有业务判断但缺少来源 | 输出补证任务 |
| E 禁止使用 | 无法验证、过期、侵权、敏感或合规风险高 | 禁用并加入风险清单 |

## 报告必补模块

- 分析完整性矩阵：说明每个维度的现状、缺口、动作和验收指标。
- 证据成熟度与事实底座：说明哪些事实可用、哪些待补证、哪些不能写。
- 预算资源与优先级：说明人力、内容产能、开发排期、预算级别和取舍逻辑。
- 治理节奏与决策机制：说明周会、双周复测、月度复盘和升级机制。
