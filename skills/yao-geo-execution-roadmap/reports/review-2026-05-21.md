# yao-geo-execution-roadmap Review - 2026-05-21

## Review Scope

- 权威参考是否足够支撑分析和生成报告的底层逻辑。
- Skill 是否强制输出系统、详细、完整的报告模块。
- HTML 可视化报告是否具备固定跟随菜单栏。
- Word/PDF/HTML/Markdown 示例报告是否重新生成并通过自检。
- Word 表格是否继续使用固定宽度布局，避免向右溢出。

## Findings And Fixes

| Finding | Fix |
|---|---|
| 原方案强调六个项目包，但缺少生成计划前的系统性分析框架。 | 新增 `analysis-completeness-framework.md`，要求十维分析矩阵：业务目标、问题覆盖、品牌实体、页面技术、内容资产、标题体系、知识库、外部证据、平台适配、监测治理。 |
| 原方案不能清晰回答“能否拿到真实数据”。 | 新增 `real-data-capability.md`，把真实数据拆成用户提供、公开网页核验、平台采样导入和缺口治理四类，并要求报告输出数据模式、采集计划、来源类型、证据等级、核验日期和新鲜度。 |
| 证据质量只在来源台账中体现，无法区分可用事实、待补证事实和禁用表述。 | 新增 A-E 证据成熟度模型，并在报告中加入“证据成熟度与事实底座”章节。 |
| 报告缺少资源、预算、治理和升级机制，容易变成任务清单。 | 新增“资源预算与优先级”“治理节奏与决策机制”章节，并把对应字段纳入接口、模板、示例输入和质检。 |
| HTML 报告长页阅读时缺少章节导航。 | 渲染器新增 `report-menu` 固定菜单栏，章节锚点覆盖所有核心二级标题，打印时自动隐藏。 |
| HTML/PDF 版式偏普通表格报告，未充分吸收 kami 的编辑排版。 | 更新 HTML/PDF CSS：白底主画布、暖白辅助面、暖灰边框、油墨蓝强调、serif 标题、sans 正文、正文行距 1.54、表格固定布局和断词防溢出。 |
| 示例报告没有覆盖新增能力。 | HubSpot 与 Lingxu 示例输入补齐十维分析矩阵、证据成熟度、资源预算、治理节奏，并重新生成四格式报告。 |
