# 交付物设计画像

## 目标读者

- 技术团队：需要可执行字段、代码片段和验收命令。
- SEO 团队：需要页面结构、schema、内链和内容信号。
- 内容负责人：需要事实补强、FAQ、摘要、案例和来源。
- 官网产品经理：需要模块级改造和优先级。
- 代理商交付团队：需要可复用报告模板和四件套。

## 报告结构

1. 执行摘要。
2. 输入、范围和无法判断项。
3. 页面组合与选择依据。
4. 公开答案素材问题集。
5. 权威证据台账。
6. 可抓取性、渲染、移动和性能诊断。
7. 结构规范性与内容信号。
8. AI 可抽取性和证据质量。
9. Schema 一致性和实体网络。
10. 代码层修复清单。
11. 内容结构改造建议。
12. Schema 与 HTML 模块建议。
13. 优先级、成本和验收计划。
14. 完整性自检与质量报告摘要。

## 交互与排版

- HTML 报告默认带目录菜单栏，滚动时固定跟随，适合长报告快速定位模块。
- 报告视觉遵循 Kami：暖纸面、油墨蓝强调、暖灰边框、中文标题 serif、正文 sans、紧凑编辑节奏。
- Word 和 PDF 以不溢出为最高优先级，所有宽表拆成 5 列以内。
- Markdown 保留证据台账、代码片段、验收命令和完整性自检，作为四件套唯一内容源。
