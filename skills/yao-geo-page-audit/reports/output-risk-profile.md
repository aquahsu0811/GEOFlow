# 输出风险画像

- 高风险：把页面 GEO 诊断写成传统 SEO 文案建议，缺少 HTML 可读性和 AI 可抽取性。
- 高风险：schema 与正文事实不一致，导致结构化数据变成事实堆砌。
- 高风险：Word 宽表、长 URL、长命令向右溢出，导致交付体验差。
- 高风险：报告结论缺少权威证据层级，把推断写成确定事实。
- 中风险：HTML 长报告没有固定目录，用户难以定位代码、内容和验收模块。
- 中风险：没有日志时误判爬虫真实召回。
- 中风险：没有 AI 平台采样时误写平台召回、排名、答案频次或引用份额。
- 中风险：只检查首页，忽略一级页、二级页、帮助中心、案例页和文档页。

## 缓解方式

- 使用五类诊断维度和 P0/P1 字段要求。
- 使用权威证据台账和系统化报告模块分类。
- 默认删除平台表现分析，仅保留公开答案素材覆盖；只有用户提供平台样本时才写平台表现。
- 所有四件套从同一 Markdown 内容源生成。
- 使用 `scripts/review_report_layout.py` 检查 Word/PDF 右溢风险和 HTML sticky TOC。
- 示例目录附带 `quality-report.json`。
