# 示例站点页面 GEO 诊断报告

> 公开合成示例。品牌：示例云服。站点：[示例官网](https://www.example.com/)。分析日期：2026-05-21。生成方式：`yao-geo-page-audit`。本报告用于展示 skill 的完整输出结构，不代表真实企业诊断结论。

## 1. 执行摘要

示例站点的 GEO 风险集中在三个层面：第一，核心正文和 FAQ 的初始 HTML 可读性不足；第二，产品事实缺少可切片引用的表格、键值对和上下文无关摘要；第三，schema 中的服务范围、价格和 FAQ 字段没有稳定回到页面正文。

优先级判断：

- P0：核心产品说明、服务边界和 FAQ 必须进入初始 HTML，不能只依赖客户端渲染。
- P0：schema 字段必须与页面正文一致，特别是价格、服务范围和 FAQ。
- P1：首页、一级页和二级页都应增加“事实卡、对比表、FAQ、更新时间、来源”模块。
- P1：HTML 报告、官网页面和 Word 交付件都要避免宽表右溢。

## 6. 可抓取性与渲染诊断

| 检查项 | 观察结果 | 风险 | 建议 | 优先级 |
|---|---|---|---|---|
| 主内容 HTML | 核心 FAQ 不在初始 HTML | AI 只能读取浅层摘要 | FAQ 和事实表 SSR/SSG 输出 | P0 |
| robots | 未见关键页面被禁用 | 风险较低 | 保留公开访问，补 sitemap | P2 |
| canonical | 栏目页 canonical 指向自身 | 风险较低 | 语言版本分别声明 | P2 |
| JS 依赖 | 产品说明部分依赖客户端渲染 | 弱渲染爬虫可能漏读 | 关键事实不要后置渲染 | P0 |
| 初始 HTML | 导航文本占比高 | 主内容抽取噪音增加 | 用 `main` 和模块 ID 标记正文 | P1 |

## 7. 移动端、性能与可访问结构

| 维度 | 观察 | 风险 | 改造建议 |
|---|---|---|---|
| 移动端表格 | 参数表在窄屏可能溢出 | 用户和爬虫预览体验差 | 表格容器横滚，移动端改字段卡片 |
| 首屏稳定 | 首屏营销图较重 | LCP 可能受影响 | 压缩图片，核心摘要先于大图输出 |
| 内容懒加载 | FAQ 可能需要交互后加载 | 移动优先索引可能漏内容 | FAQ 默认进入 HTML |
| 标题结构 | H2/H3 顺序基本清晰 | 个别模块标题语义弱 | 每页仅一个 H1，H2 描述主题 |
| 链接文字 | 存在“了解更多”类锚文本 | AI 难判断链接目标 | 锚文本写清产品或问题 |

## 8. 结构规范性与内容信号

| 模块 | 当前问题 | GEO 影响 | 修复方向 |
|---|---|---|---|
| 页面摘要 | 缺少 80-120 字上下文无关摘要 | AI 引用开头不稳定 | 首屏增加实体全称、对象、价值和边界 |
| 产品事实 | 功能点分散在段落 | 难以切片回答 | 增加字段化事实表 |
| 服务边界 | 未说明不适用场景 | 答案容易过度承诺 | 增加适用/不适用表 |
| 案例证据 | 缺少可引用客户案例 | 信任信号不足 | 增加案例、行业、结果和来源 |
| 更新时间 | 页面未突出更新时间 | 新鲜度弱 | 每个事实模块显示更新时间 |

## 9. AI 可抽取性诊断

| 信号 | 判断 | 风险 | 修复方向 |
|---|---|---|---|
| 原子事实 | 不足 | AI 难以按问题切片引用 | 建立事实字段、值、来源表 |
| 上下文无关摘要 | 不足 | 摘要脱离标题后意义不完整 | 每页首屏加入 80-120 字摘要 |
| 对比表 | 缺失 | 竞品问题易引用第三方 | 增加对比维度和适用边界 |
| 步骤内容 | 不足 | “如何落地”问题缺素材 | 增加角色、步骤、输入和输出 |
| 证据质量 | 中 | 缺少作者、来源和更新时间 | 补充负责人、数据来源和案例边界 |

## 10. Schema 一致性诊断

| 类型 | 当前状态 | 风险 | 建议 |
|---|---|---|---|
| Organization | 可建立品牌实体 | sameAs 和官方资料不足 | 增加官网、百科、社媒和联系页 |
| WebPage | 可描述页面 | 摘要字段弱 | 与页面摘要保持一致 |
| Product | 字段不完整 | 功能和价格边界难回溯 | 先补正文事实，再补 schema |
| FAQPage | FAQ 正文不足 | schema 可能脱离用户可见内容 | FAQ 必须页面可见 |
| BreadcrumbList | 部分页面缺失 | 层级关系弱 | 栏目和详情页补面包屑 |

## 13. Schema 与 HTML 模块建议

```html
<main id="main-content" data-geo-main="true">
  <section id="geo-summary" aria-labelledby="geo-summary-title">
    <h2 id="geo-summary-title">页面摘要</h2>
    <p>示例云服是面向 B2B 制造企业的订单协同平台，帮助销售、计划和生产团队同步订单进度、交付风险和客户沟通记录。</p>
  </section>
</main>
```

```json
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [
    {
      "@type": "Question",
      "name": "示例云服适合哪些企业？",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "示例云服适合需要统一订单进度、生产计划和客户沟通记录的 B2B 制造企业。"
      }
    }
  ]
}
```
