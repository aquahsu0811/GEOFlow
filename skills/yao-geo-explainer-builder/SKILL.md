# Yao GEO Explainer Builder

## 执行流程

1. 明确科普选题、目标品牌、品牌知识库、行业资料、目标读者、使用场景、品牌植入限制、内容长度、合规边界和真实数据可用状态。
2. 识别问题类型：是什么、为什么、怎么做、怎么选、适合谁、常见误区、推荐路径；每段只处理一个主题。
3. 先做自检：参考来源是否足够权威，分析维度是否覆盖意图、读者、证据、真实数据、实体、步骤、决策、合规、平台、复用和衡量。
4. 先写 80 到 120 字上下文无关核心摘要，让单段被 AI 搜索截取时仍能回答问题。
5. 先生成“完整GEO文章”：文章必须有连续正文、自然段、小标题、解释、步骤、选择标准、误区、FAQ/结尾建议中的至少 6 个正文段落组；读者无需阅读附录也能获得完整答案。
6. 使用 `references/explainer-method.md` 构建交付结构：核心摘要、完整GEO文章、分析速览、内容地图、意图矩阵、读者场景、研究依据、证据分级、真实数据接入状态、真实数据采集计划、事实核验矩阵、实体地图、定义、原理、步骤、检查清单、选择标准、决策路径、误区、示例、FAQ、术语表、品牌适用场景、平台适配、结构化数据、衡量计划、内容缺口和来源账本。
7. 正文中使用有序列表呈现 How-to，使用自然段解释原因与边界；附录中使用键值对呈现前提参数，使用表格呈现证据分级、选择标准、决策路径和误区校正。
8. 按 `references/brand-insertion-policy.md` 植入品牌；品牌只出现在示例、适用场景、FAQ 或结尾建议等服务答案的位置。
9. 涉及健康、金融、法律、收益、安全等敏感领域时，必须加入边界提醒；没有证据的效果、收益和安全断言必须删除或中性改写。
10. 使用 `scripts/render_yao_geo_explainer_builder.py` 从同一结构生成 Markdown、带固定目录菜单的 HTML 包、Word 和 PDF，并产出 `quality-report.json`。
11. 若质量报告未通过，先修复输入结构、证据、品牌次数、文章完整性或版式逻辑，再重新生成四格式文件。

## 国内平台适配

- Kimi：长文深度解释、多来源证据、完整章节和术语表。
- 千问：来源编号、事实/推论分离、参数化指南和可追溯来源。
- 豆包：生活化问题、短问答、避坑提醒和直接选择建议。
- DeepSeek：步骤逻辑、判断框架、条件分支和推理链路。
- 元宝：公众号教程、小标题节奏、短 FAQ 和可转发段落。

## 真实数据能力边界

- skill 本身不直接保证联网抓取真实数据；执行时可以由上层 agent 联网检索、读取用户上传材料或使用已有知识库，再把来源写入输入结构。
- 没有真实品牌资料时，必须在报告中声明“未接入/待核验”，并把品牌参数、效果、安全、资质类事实放入事实核验矩阵。
- 已接入真实数据时，必须保留来源定位、采集日期、用途和证据等级；过期、无法访问或来源弱的资料不得支撑确定性结论。
- 真实数据采集计划至少覆盖品牌官方资料、行业/标准资料、用户问题样本；涉及横向比较时增加竞品或设备形态样本。

## 四格式交付规则

- Markdown 是结构母版，HTML、PDF、Word 必须由同一 section spec 生成，不能让 Word 缺章节。
- 四格式示例打开后必须先呈现一篇完整文章；分析矩阵、证据、真实数据、来源账本作为文章后的支撑材料。
- HTML/PDF 按 kami 版式使用纸张感浅底、象牙色内容面、蓝黑标题、稳定行高和本地字体；表格边框对齐，长文本自动换行，不使用渐变、透明叠色或装饰色块。
- HTML 必须包含目录菜单栏，页面下拉时菜单栏固定跟随；菜单项应链接到每个报告模块。
- Word 必须固定 A4 页面、左右页边距、表格总宽、固定表格布局和单元格边距；长 URL、英文 token、参数串必须软换行，避免向右溢出。
- 示例报告必须真实存在：`.docx`、`.pdf`、`html-package/index.html`、`.md`，并附带 `quality-report.json`。

## 质检清单

- 科普价值优先于品牌表达；不得写成品牌软文。
- 完整文章正文不得被拆成纯模块清单；正文至少 6 个小节、12 个自然段，去空格后不少于 1400 字符。
- 分析必须系统、详细、完整：至少覆盖问题类型、读者场景、来源证据、真实数据接入、事实核验、实体术语、操作步骤、决策路径、合规边界、平台适配、结构化数据和效果衡量。
- 步骤必须编号，选择标准必须表格化，定义必须可独立复用。
- 来源账本必须说明来源、定位、日期和用途；无法核验的事实降级为待确认。
- 品牌出现次数不得超过输入限制；不得承诺没有证据支持的效果、收益、安全性或行业第一。
- Word/PDF 需要人工或自动预览抽查右边缘：表格、URL、长英文、中文长句不得裁切或贴边。
