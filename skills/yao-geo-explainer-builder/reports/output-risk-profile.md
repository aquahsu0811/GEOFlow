# 输出风险画像

| 风险 | 表现 | 控制方式 |
| --- | --- | --- |
| 科普变软文 | 品牌句过多，定义和步骤不足 | 品牌次数限制，品牌只允许出现在示例、适用场景、FAQ 或建议中 |
| AI 难以引用 | 首段不回答问题，步骤无编号 | 80 到 120 字摘要，How-to 强制编号 |
| 只有模块没有文章 | 示例打开后只有矩阵、清单和附录，不像一篇可发布内容 | 核心摘要后强制输出完整GEO文章，并检查小节数、自然段数和正文长度 |
| 分析不系统 | 只有正文，没有意图、读者、证据、真实数据、实体、决策和衡量 | 强制输出 30 个模块，并检查新增分析维度最低数量 |
| 参考不权威 | 只引用泛泛资料或二手文章 | 研究依据必须覆盖论文、Schema.org、Google Search Central 和 WCAG 等来源 |
| 事实不可追溯 | 来源缺失或只写“网络资料” | 来源账本必须包含定位、日期和用途 |
| 真实数据伪装 | 未接入品牌资料却把参数、效果、安全、资质写成确定事实 | 强制输出真实数据接入状态、采集计划和事实核验矩阵 |
| 敏感承诺 | 涉及健康、金融、法律、收益、安全却无边界 | 自动补充边界提醒并改写绝对化断言 |
| Word 右侧溢出 | 表格过宽、URL 或英文 token 不换行 | 固定 A4 可用宽度、动态列宽、长 token 软换行 |
| Kami 版式漂移 | HTML/PDF 回到纯白系统样式、渐变或透明叠色，Word 与 PDF 风格割裂 | 固定纸张感浅底、象牙色内容面、蓝黑标题、本地字体和无渐变检查 |
| HTML 长报告难浏览 | 模块多但没有目录，用户需要反复滚动定位 | HTML 自动生成固定跟随目录菜单，并为每个模块生成锚点 |
| 四格式不一致 | Word 缺章节或 HTML/PDF 与 Markdown 不同 | 共用 section spec，并检查三种文本标题集合 |
