# 评估 Rubric

| 维度 | 通过标准 |
| --- | --- |
| 路由准确 | 命中科普、How-to、怎么选、避坑、FAQ、术语表和四格式交付请求 |
| 完整文章 | 核心摘要后先输出一篇可发布的 GEO 文章，至少 6 个小节、12 个自然段，读者无需看附录也能完成理解和决策 |
| 内容结构 | 文章正文中的定义、原理、步骤、标准、误区、示例、FAQ 或结尾建议齐全，附录中来源账本齐全 |
| 分析完整度 | 意图矩阵、读者场景、证据分级、实体地图、决策路径、结构化数据、衡量计划和内容缺口齐全 |
| 真实数据 | 明确当前是否接入真实数据，列出缺失数据、采集计划和事实核验矩阵 |
| GEO 适配 | 信息型与多跳推理问题可被直接截取，来源和条件清晰 |
| 品牌客观性 | 品牌植入服务答案，不破坏科普客观性 |
| 合规 | 敏感领域有边界提醒，绝对化断言被中性化 |
| 四格式 | Word、PDF、HTML 包、Markdown 全部存在且结构一致 |
| 排版 | 符合 kami 纸张感版式，表格对齐、行高稳定，HTML 有固定跟随目录菜单，Word/PDF 无右侧溢出 |
