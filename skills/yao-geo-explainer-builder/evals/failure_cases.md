# 失败样例

## 品牌软文

如果正文大量重复目标品牌，缺少定义、步骤和来源，应判定失败。

## 步骤不可复用

如果 How-to 只写成一段散文，没有编号、动作和检查点，应判定失败。

## 只有报告模块没有文章

如果示例打开后只是分析表、矩阵、清单和附录，缺少一篇连续、可发布、读者可直接阅读的完整 GEO 文章，应判定失败。

## 证据不足

如果声称“最安全”“100% 有效”“行业第一”但没有来源，应删除或改为条件化表达。

## 分析不完整

如果报告只有正文和 FAQ，没有问题类型、读者场景、证据分级、实体地图、决策路径、结构化数据、衡量计划和内容缺口，应判定失败。

## 真实数据伪装

如果报告没有说明当前是否接入真实品牌资料，或把未核验的品牌参数、效果、安全、资质写成确定事实，应判定失败。

## 四格式漂移

如果 Word 缺少 Markdown/HTML 中的章节，或 PDF 不是从同一 HTML 生成，应判定失败。

## Kami 版式漂移

如果 HTML/PDF 没有使用纸张感浅底、象牙色内容面、蓝黑标题、本地字体和稳定行距，或出现渐变、透明叠色导致可读性下降，应判定失败。

## HTML 缺少固定菜单

如果 HTML 长报告没有目录菜单，或菜单不能在页面下拉时固定跟随，应判定失败。

## Word 溢出

如果表格总宽超过正文宽度，或长 URL、英文参数串在 Word 中向右裁切，应判定失败并修复渲染逻辑。
